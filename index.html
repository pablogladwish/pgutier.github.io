What is HTML?

HTML is an acronym for hypertext markup language.

HTML is the language (well, techincally it is only a markup [a way to tag information] and not a language) that is used for making web pages. It is the framwork of web pages. It is structural. It is semantic (meaning-based). It has some basic design inherently built into it. For example. This paragraph is under an <h2>heading tag. That tag and all of the <h1> to <h6> heading tags create visual heirarchy within text. Notice the size waterfall <h1> to <h6> tags below. This allows one to make many sub category headings within text.

This paragraph is written within a <p> tag, which represents the paragraph. Paragraphs can take headings if needed.

Browsers interpret HTML and structure pages so they have the intended visual heirarchy that reflects to the sematics (meaning) of the content. HTML also allows search engines to understand what is on the pages so writing correct HTML is important. Without it your work does not fit into the information ecosystem of the web correctly and search engines will not like your website.

Pure HTML is inherently responsive (scales with the browser window). As long as you use relative units of measurements (percentages and ems) then you can make simple responsive pages. Percentages are pretty easy to understand. EMs are relative to the brower preferences. If the browser is set to display the body text at 14pt, then one EM is equal to 14pt.

Later we will learn how to alter the appearance HTML tags with CSS (Cascading Style Sheets). This is where you can make more unique design decisions.

Below is an image. This image is contained within this website folder. The code for it is a defined pathway. It looks like this. <img src="images/web-picture.jpg" alt="This is a black and white image of a chess game in a park. All of the pieces have fallen over."/> The “images” is a folder within the site folder. The slash represents that the file "web-picture.jpg" is within the images folder. This is important to understand as students often make the mistake of not keeping track of the images they use in their site. All the images in the site should be contained within one images folder and referenced correctly in the code. If the images are not where the code says they are then the pathways are not possibly and images do not show up on pages. Also if images are too big (in terms of file size) then they will not load quickly). Read the comments in the code for more information on images. Comments in HTML are always written in this way <!--This is a comment. It is not active code It is just notes for other coders to share how things work.-->.
